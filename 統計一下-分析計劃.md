## 三、Preset 切分策略

**結論：一個類別一個 preset**，理由三點：
1. 每次跑出的 `channel_videos_<ts>.csv` 對應一個 preset，分析時只要載入單一檔案即可，notebook 邏輯最乾淨（不需要在 pandas 裡再做類別篩選）。
2. 不同類別 Shorts 豐度差很多，`max_videos_per_channel` 需要不同設定（見下節）。
3. 配額單次 8000 上限放不下全部 9 類；逐類跑可以避免「跑到一半 budget exceeded」。

**範例設定（不寫 yaml，只列數值）**

| Preset | channel_ids | max_videos_per_channel | 配額估算 |
|---|---|---|---|
| sports | @NBA, @MLB, @挖掘肌讲健身, @老赵闹健身 | 300（NBA/MLB 為長片為主，要拉高） | 4 handle + ceil(1200/50)≈24 page + ceil(1200/50)≈24 ≈ **52 units** |
| pansci | @PanScitw | 200 | 1 + 4 + 4 ≈ **9 units** |
| games | @camman18, @GothamChess | 200 | 2 + 8 + 8 ≈ **18 units** |
| news | @shasha77, @johnnyharris, @Vox, @SubsOverflow, @RealFake | 200 | 5 + 20 + 20 ≈ **45 units** |
| basketball_meme | @junhongtv, @juanmaodonggeqiu | 200 | ≈ **18 units** |
| basketball_influencer | @肖他11, @ehoozeng | 200 | ≈ **18 units** |
| food_review | 7 個 @handle | 200 | 7 + 28 + 28 ≈ **63 units** |
| kpop | @kukuo1019, @k-chaoliu, (Lablodol?) | 200 | ≈ **27 units** |

> 公式：`1*ceil(channels/50) + 1*pages_per_uploads*channels + 1*ceil(total_videos/50)`，每個 handle 解析另外 +1 unit（`resolve_channel_ids` 對每個 @handle 都 `channels.list`）。

**配額結論**：9 個類別總和約 250 units，遠低於單日 10,000 預設配額；可以**一天內全部跑完**。不需要排程跨天。

## 四、針對「特殊類別」的處理建議

### 1. 科普（單頻道 @PanScitw）

不能跑頻道因子 ANOVA、沒有跨頻道 OLS dummy。改用以下「單頻道變體」：

- **時間分段 ANOVA**：把 `published_at` 切成 3–4 個區間（例：每季、每半年、最近 30 天 vs 之前），檢驗「不同期間發布的 Shorts 觀看數平均是否有差」
- **發布時段 ANOVA**：把 `pub_hour` 切早/中/下/晚/深夜五組（這個 Reg/02 已經有，但僅作描述；單頻道時把它升級成主分析）
- **保留所有「視訊內變數」分析**：title_length、tag_count、duration_sec、相關性、迴歸（model1 仍可跑，只是沒有 channel dummy 的 model2）
- **新增「對 PanSci 自己的 baseline 比較」**：用 Shorts vs 長片（duration_sec > 180）做兩樣本檢定，回答「Shorts 是不是真的能突破長片觸及」

這條已經有 `notebooks/01_pansci_shorts.ipynb` 雛形（在主 notebooks/ 而非 Reg/），可直接搬到 `analysis/pansci/analysis.ipynb` 並升級。

### 2. Lablodol（Kpop，沒給 handle）

**建議三步驟**
1. 規劃階段：先在 Kpop preset 裡**先放兩個有 handle 的**（@kukuo1019, @k-chaoliu），跑出資料。
2. 在 YouTube 上手動搜「Lablodol」找到頻道頁，從 URL 拿 `UC...` channel_id 或 `@handle`，補進 preset 再跑一次。
3. 若實在找不到 → Kpop 走 two-channel 模板，並在 `findings.md` 註明排除原因。

### 3. NBA / MLB / Vox / Johnny Harris：Shorts 樣本稀少風險

這些頻道是長片主導，`max_videos_per_channel=200` 可能只抓到個位數 Shorts（duration_sec ≤ 180）。

**建議**
- 運動類把 `max_videos_per_channel` 拉到 **300–500**
- 時事類同理拉到 **300**
- 跑完後先在 notebook 第一格印出「Shorts 篇數 by channel」；若某頻道 < 30 篇，在 findings.md 明確標示為「樣本不足，僅作描述」，不納入 ANOVA / 迴歸
- 必要時把 Shorts 定義放寬：Reg/03 已用 `duration_sec <= 180`（含新版較長 Shorts），延續這個閾值

- 多頻道箱線圖、Top 10 比較
- 加入 §4.5 內容貼標 + §7.5 內容標籤分析

### D. `_crosscat/cross_category.ipynb`（跨類別總整合）

**這是企劃書沒明寫但很值得做的「最高層級」分析**。把全部 9 個類別合併成一個 DataFrame，加上 `category` 欄位。研究問題對應如下：

- **跨類別 ANOVA**：不同類別的 Shorts 觀看數中位數 / 平均是否顯著不同 → 直接回答「教育類 vs 娛樂類演算法權重」
- **正規化技巧**（解決 NBA 千萬訂閱 vs Lisa Sung 6 萬的不公平問題）：
  - 主要指標改用 `view_per_subscriber = view_count / subscriber_count`（觸及率）
  - 或對 `view_count`、`subscriber_count` 取 `log10` 再做 OLS
- **大 OLS**：`log(view_count) ~ log(subscriber_count) + title_length + tag_count + duration_sec + C(category)`
- **類別 × 變數 交互作用**：例如「標籤數量對觀看的影響，在不同類別是否不同」（formula 加 `C(category) * tag_count`）
- **YouTube 自動 topic 矩陣**：行 = 使用者手動類別，列 = YouTube topic，cell = 命中率 → 答「使用者的人工分類與 YouTube 演算法視角一致嗎」
- **跨類別內容相似度**：所有頻道兩兩 TF-IDF cosine → 熱圖；按使用者分類排序 → 看「同類別內聚」程度
- **LDA 主題模型**：在合併後的 ≥ 2000 部影片標題上跑 LDA，找出 8–12 個 latent topics，再對照使用者分類觀察

## 七、內容分析 + 自動貼標（補強）

> 對應企劃書研究背景：「教育類 vs 娛樂類因留存與資訊價值差異而獲不同演算法權重」假設。

### 根本權衡：手上沒有「影片內容」，只有四種文字訊號

YouTube Data API 不給逐字稿，現有 CSV 能用的文字訊號：

| 訊號 | 來源 | 覆蓋率 | 訊息密度 |
|---|---|---|---|
| `title` | 影片標題 | 100% | 高 |
| `tags` | API tags 欄（`\|` 串接） | ~50%（Reg/02 顯示 211/410 為 NaN） | 中 |
| `description` | 影片簡介 | 低（Reg/02 顯示 209/410 為 NaN） | 高但稀疏 |
| `topic_categories` | YouTube 自動分類的 Wikipedia URL | ~95% | **每部都有 1–3 個主題**（免費！） |

**關鍵發現**：`topic_categories` 在 Reg/02 範例已經出現 `https://en.wikipedia.org/wiki/Food`、`Entertainment`、`Lifestyle` — YouTube 自己已經幫每支影片貼好 high-level 標籤了，**這是被低估的免費資源**。

### 四層貼標策略（由廉到貴，可疊加）

#### Layer 1：YouTube `topic_categories` 解析（免費，必做）

從 Wikipedia URL 抽尾段就是 topic 名稱。直接拿來：
- 頻道級：算每個頻道的 topic 分佈（如 @Ju茱莉亞林 = Food 80% / Entertainment 60% / Lifestyle 40%）
- 驗證手動分類：對照「使用者把 @PanScitw 歸類為科普」vs「YouTube 標為 Knowledge / Science」是否一致
- 跨類別比較：不同類別在 YouTube 系統內被標的 topic 重疊度

#### Layer 2：規則式關鍵字字典貼標（必做，企劃書 RQ #2）

為每個類別**預先寫一張關鍵字字典**，每部影片 0/1 標：

```text
教學/教育：教學, 教程, 怎麼, How to, tutorial, ...
挑戰：挑戰, challenge, vs, 對決, ...
搞笑/迷因：搞笑, 整人, meme, 笑死, ...
開箱/評測：開箱, 測試, review, 試吃, ...
互動誘導：你, 留言, comment, 猜, ...
情緒詞：震驚, 不可思議, 居然, 竟然, OMG, ...
品牌/推廣：（針對「餐廳/食品推廣」類）廠商, 合作, 業配, 雅方, ...
```

**為什麼這比 NLP 模型更適合期末報告**：
- 可解釋（統計課要的就是這個）
- 可重複（同一字典套到所有類別）
- 適合卡方檢定、logistic regression（直接答 RQ #2）

#### Layer 3：TF-IDF 描述性內容指紋（建議做）

對每個頻道的標題語料做：
- **Jieba 中文分詞**（必須裝，pure 字元 n-gram 對中文沒意義）
- TF-IDF 取每個頻道 Top 15 keywords
- 視覺化：詞雲 / 條形圖
- 跨頻道：頻道兩兩 cosine similarity，畫熱圖

**輸出**：每個頻道一張「內容指紋卡」放進 `content_profile.md`：

```
@Ju茱莉亞林 內容指紋
─────────────────────
YouTube topics: Food | Entertainment | Lifestyle
Top keywords (TF-IDF): 雅方, 國際, 茱莉亞林, 羊肉爐, 冰淇淋, 藏鏡人, 三兄妹, ...
規則標籤命中率：品牌推廣 95% | 家庭互動 62% | 食物 87%
標籤類型分佈：產品標籤 (#雅方羊肉爐) 70% / 品牌標籤 (#茱莉亞林) 30%
平均標題長度：CJK 字 12.3 / EN 字 0.1
```

#### Layer 4：LDA 主題模型（選做，類別內 sub-themes）

每個類別內部跑 LDA，找 3–5 個 latent topics。
- 例：「介紹食物」7 個頻道 → 可能跑出 [日式美食、街邊小吃、餐廳評測、減肥餐、亞洲料理] 等 sub-topics
- 每個 sub-topic 對應「相對觀看數」→ 可答「同類內容中，哪種次主題更易爆」

**不建議用 LLM 貼標**：在統計課裡用 LLM 黑箱貼標難以辯護方法論；報告書答辯時老師問「你怎麼定義這個標籤」會卡住。Layer 2 + Layer 3 已足夠。

### Notebook 模板新增區塊

**§4.5 內容貼標**（插在原 §4「資料清理」與 §5「描述統計」之間）
```
- Layer 1: parse topic_categories → 頻道 topic 分佈表 + 堆疊條圖
- Layer 2: apply keyword dict → 11 個 0/1 欄位
- Layer 3: jieba + TF-IDF → 每頻道 Top 15 keywords + 詞雲
- 輸出：每個頻道一張內容指紋卡（存到 content_profile.md）
```

**§7.5 內容標籤與觀看數的關係**（接 §7 相關性之後）
```
- 卡方檢定：每個 keyword 標籤 vs 高觀看（top quartile）
- Logistic regression: P(top quartile) ~ has_教學 + has_挑戰 + has_搞笑 + ...
- 視覺化：每個標籤命中組 vs 未命中組的觀看數箱線圖
```

### 為什麼這對企劃書是巨大加分

| 企劃書 RQ | 原規劃 | 加入內容分析後 |
|---|---|---|
| #2 標題關鍵字 | 簡化版 0/1 | **完整版**：Layer 2 字典 + 卡方 + logistic |
| #4 標籤類型 | 中優先級 | Layer 1（YouTube topic）+ Layer 3（TF-IDF）直接回答 |
| #5 標題-標籤一致性 | 低優先級降為 Jaccard | Layer 2 命中於 title vs 命中於 tags 的一致率 |
| **研究背景假設**：「教育類因高留存獲長尾推撥」 | 沒回答 | **跨類別 ANOVA 直接驗證** |

研究背景那段「教育類 vs 娛樂類演算法權重」假設，原規劃靠跨類別 ANOVA 只能比「觀看數」；加上內容標籤後可以**比較同類別內被標為「教學」vs「未被標為教學」的觀看差**，這比類別本身更有解釋力（因為類別還混了訂閱數效應）。

## 九、風險清單與處理建議

| # | 風險 | 處理 |
|---|---|---|
| 1 | **長片頻道 Shorts 樣本太少**（NBA / MLB / Vox / Johnny Harris） | 把 `max_videos_per_channel` 拉到 300–500；notebook 第一格印 channel-level Shorts 篇數；< 30 篇者僅描述不檢定 |
| 2 | **中英文字元數計算不公平** | 在所有 notebook 額外加 `title_length_en`（用 regex 拆 ASCII 字數）與 `title_length_cjk`（CJK 字元數），跨類別比較時偏好用 CJK 計數；報告書註明本研究以「視覺上字符數」為準 |
| 3 | **訂閱數量級差異**（NBA 千萬 vs Lisa Sung 6 萬） | 跨類別分析強制用 `log10(view_count)` + `log10(subscriber_count)`，並用 `view_per_subscriber` 作為次要指標 |
| 4 | **handle 解析失敗**（中文 handle、Lablodol 無 handle） | 跑 preset 前先用 `python main.py channel-compare --preset <name> --dry-run`，再小規模實跑 `max_videos_per_channel: 5` 驗證 channels.list 回傳是否完整；Lablodol 手動找 channel_id 補回 |
| 5 | **Reg/02、Reg/03 的 CSV 時間戳寫死在 notebook 裡** | 模板第一格加一個 `CSV_TIMESTAMP = "20260527-XXXXXX"` 常數，下方所有 `pd.read_csv` 都引用它；改類別時只改這一行 |
| 6 | **配額耗盡**（極端情況：某頻道上傳數遠超 max_videos_per_channel 設定） | `paginate(..., max_pages)` 已限制；保持 `max_videos_per_channel` 在 500 以內 |
| 7 | **資料抓取與分析時間不同步**（隔天重跑 channel-compare 會多新時間戳 CSV） | 在每個類別資料夾的 `DATA_REF.md` 寫明分析用的 CSV 時間戳；如要重抓，notebook 只改 `CSV_TIMESTAMP` 一行 |
| 8 | **跨類別合併時 channel_title 撞名** | 合併用 `channel_id` 當鍵，不用 channel_title；category 欄位用 preset 名稱（程式碼裡用一張映射表） |
| 9 | **`src/common.py` 的 `flatten_video` 中 tags 用 `\|` 串接** | notebook 算 `tag_count` 時要用 `df['tags'].str.split('\|').str.len()`，注意 None / NaN |
| 10 | **Reg/02 ANOVA 對 title_length 切點寫死 [0,15,30,100]** | 跨類別時改用 `pd.qcut` 三等分位點，避免某類別全部標題都 <15 字導致空組 |
| C1 | **中文簡繁、日文標題混雜** → jieba 切錯 | 用 `opencc` 統一轉繁；日文（うまぐるめ）改用 char n-gram |
| C2 | **關鍵字字典帶主觀偏誤** | 字典 yaml 化、附「字典版本」到 findings.md、報告書方法論段註明 |
| C3 | **Tags 半數 NaN** → Layer 2 在 tags 上命中率低 | Layer 2 主要跑在 title；tags 命中當輔助訊號 |
| C4 | **topic_categories 是英文 Wikipedia** → 對中文小眾頻道可能缺值 | 缺值率 < 10% 可接受；缺值單獨歸為 "uncategorized" 一類 |
| C5 | **LDA 在小樣本（單頻道、200 部）不穩定** | LDA 只在跨類別 ≥ 500 部的場景跑；類別內樣本 < 300 跳過 Layer 4 |
